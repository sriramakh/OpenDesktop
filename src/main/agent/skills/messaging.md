# Messaging Skill Guide (Slack + Microsoft Teams)

Last verified: 2026-04-06

---

## Section 1: When to Use

Use the messaging tools when the user says:
- "send a message to Slack", "post to #channel", "notify Slack"
- "send to Teams", "post to Teams", "message Teams channel"
- "search Slack for X", "find messages about X in Slack"
- "send a formatted message", "send a rich card"
- "send a status update", "notify the team"

**5 tools total**: 3 Slack + 2 Teams.

## Section 2: Tool Reference

### Slack Tools

| Tool | Permission | Required Params | Optional Params | Auth Required |
|------|-----------|----------------|----------------|---------------|
| `slack_send` | sensitive | `message` | `webhookUrl`, `channel`, `username`, `iconEmoji` | `slack_webhook` in KeyStore |
| `slack_send_blocks` | sensitive | `blocks` | `text`, `webhookUrl`, `channel` | `slack_webhook` in KeyStore |
| `slack_search` | safe | `query` | `count` (default: 10) | `slack_bot_token` in KeyStore |

### Teams Tools

| Tool | Permission | Required Params | Optional Params | Auth Required |
|------|-----------|----------------|----------------|---------------|
| `teams_send` | sensitive | `message` | `webhookUrl`, `title`, `themeColor` | `teams_webhook` in KeyStore |
| `teams_send_card` | sensitive | `card` | `webhookUrl` | `teams_webhook` in KeyStore |

## Section 3: Setup

### Slack Incoming Webhook

1. Go to https://api.slack.com/apps > Create New App > From scratch.
2. Under "Incoming Webhooks", toggle ON, then "Add New Webhook to Workspace".
3. Select the target channel and authorize.
4. Copy the webhook URL (format: `https://hooks.slack.com/services/T.../B.../...`).
5. In OpenDesktop: Settings > Integrations > set key `slack_webhook` to the URL.

### Slack Bot Token (for search)

1. In the same Slack App, go to "OAuth & Permissions".
2. Add the `search:read` scope under Bot Token Scopes.
3. Install/reinstall the app to the workspace.
4. Copy the Bot User OAuth Token (starts with `xoxb-`).
5. In OpenDesktop: Settings > Integrations > set key `slack_bot_token` to the token.

### Microsoft Teams Incoming Webhook

1. In a Teams channel, click the `...` menu > Connectors (or Workflows).
2. Add "Incoming Webhook", give it a name and optional icon.
3. Copy the generated URL (format: `https://outlook.office.com/webhook/...`).
4. In OpenDesktop: Settings > Integrations > set key `teams_webhook` to the URL.

## Section 4: Procedures

### Procedure: Send a Plain Slack Message

1. Call `slack_send` with `message` set to the text content.
2. The message supports Slack mrkdwn format: `*bold*`, `_italic_`, `` `code` ``, `>quote`, `~strikethrough~`.
3. Optionally override `channel` (e.g., `#alerts`), `username`, or `iconEmoji` (e.g., `:robot_face:`).
4. Result: `{ ok: true, message: "Slack message sent" }`.

Example:
```
slack_send({
  message: "*Deploy complete* :white_check_mark:\nVersion 2.4.1 is live on production.",
  channel: "#deployments",
  username: "DeployBot",
  iconEmoji: ":rocket:"
})
```

### Procedure: Send Rich Formatted Slack Message (Block Kit)

1. Build blocks using Slack Block Kit layout format (https://api.slack.com/block-kit).
2. Call `slack_send_blocks` with the `blocks` array. Include `text` as accessible fallback.
3. Common block types: `section` (text), `divider`, `actions` (buttons), `header`, `image`, `context`.

Example -- status report with sections:
```
slack_send_blocks({
  text: "Weekly Status Report",
  blocks: [
    { "type": "header", "text": { "type": "plain_text", "text": "Weekly Status Report" } },
    { "type": "divider" },
    { "type": "section", "text": { "type": "mrkdwn", "text": "*Revenue:* $142,500 (+12%)\n*Active Users:* 8,340\n*Uptime:* 99.97%" } },
    { "type": "section", "text": { "type": "mrkdwn", "text": "*Top Issue:* Database latency spikes during peak hours" } },
    { "type": "context", "elements": [{ "type": "mrkdwn", "text": "Generated by OpenDesktop | April 3, 2026" }] }
  ]
})
```

### Procedure: Search Slack Message History

1. Call `slack_search` with a `query` string. Use Slack search syntax for precision.
2. Optionally set `count` (max results, default 10).
3. Returns: `{ total, results: [{ text, user, channel, timestamp, permalink }] }`.
4. Results are truncated to 500 chars per message text.

Slack search operators:
- `from:@username` -- messages from a specific user
- `in:#channel` -- messages in a specific channel
- `has:link` -- messages containing a link
- `before:2026-03-01` / `after:2026-01-01` -- date filters
- `"exact phrase"` -- exact match

Example:
```
slack_search({ query: "deploy failure in:#incidents after:2026-03-01", count: 5 })
```

### Procedure: Send a Teams Message

1. Call `teams_send` with `message` text.
2. Optionally set `title` (card header) and `themeColor` (hex without #, e.g., `FF0000` for red).
3. The message is sent as a MessageCard with the text in the `activityText` field.
4. Default theme color is `0076D7` (Teams blue).

Example:
```
teams_send({
  message: "Build #482 passed all tests. Ready for release.",
  title: "CI/CD Update",
  themeColor: "00CC00"
})
```

### Procedure: Send a Teams Adaptive Card

1. Build an Adaptive Card JSON object (https://adaptivecards.io/designer/).
2. Call `teams_send_card` with the `card` object.
3. The card is wrapped in the Teams attachment envelope automatically.

Example -- approval card:
```
teams_send_card({
  card: {
    "$schema": "http://adaptivecards.io/schemas/adaptive-card.json",
    "type": "AdaptiveCard",
    "version": "1.4",
    "body": [
      { "type": "TextBlock", "text": "Expense Approval Required", "weight": "Bolder", "size": "Medium" },
      { "type": "FactSet", "facts": [
        { "title": "Submitted by", "value": "Jane Smith" },
        { "title": "Amount", "value": "$2,450.00" },
        { "title": "Category", "value": "Travel" }
      ]}
    ]
  }
})
```

### Procedure: Cross-Platform Notification

To notify both Slack and Teams simultaneously:

1. Call `slack_send` with the message text.
2. Call `teams_send` with the same message text (adjust formatting -- Teams uses HTML in MessageCard, Slack uses mrkdwn).
3. Both calls are independent and can be issued in sequence.

## Section 5: Known Issues & Gotchas

### Webhook vs Bot Token

- **Webhook** (`slack_webhook`, `teams_webhook`): One-way push only. Cannot read messages or search. Used by `slack_send`, `slack_send_blocks`, `teams_send`, `teams_send_card`.
- **Bot Token** (`slack_bot_token`): Two-way API access. Required ONLY for `slack_search`. Needs the `search:read` OAuth scope.
- These are separate credentials -- having one does not give you the other.

### Webhook URL Validation

- All webhook URLs must use HTTPS. HTTP URLs are rejected.
- URLs configured in the KeyStore are trusted. Inline `webhookUrl` overrides are also accepted but must be HTTPS.
- Allowed domains: `hooks.slack.com`, `outlook.office.com`, `outlook.office365.com`, `webhook.site` (testing).

### Rate Limits

- **Slack webhooks**: 1 message per second per webhook. Burst above that returns HTTP 429.
- **Slack search API**: Tier 2 rate limit (~20 requests per minute).
- **Teams webhooks**: 4 messages per second per connector. Returns HTTP 429 on throttle.
- All tool calls have a 30-second timeout.

### Slack Block Kit

- The `blocks` parameter must be a valid JSON array of block objects, not a string.
- Always include a `text` fallback for notifications/accessibility -- it is shown in push notifications and screen readers.
- Maximum 50 blocks per message.

### Teams Adaptive Cards

- The `card` parameter must be a valid Adaptive Card JSON object, not a string.
- OpenDesktop auto-wraps the card in the required Teams `attachments` envelope -- do NOT include the envelope yourself.
- Teams supports Adaptive Card schema up to version 1.5.

### Channel Override (Slack)

- The `channel` parameter in `slack_send` and `slack_send_blocks` overrides the default webhook channel.
- Format: `#channel-name` or a channel ID.
- The webhook must have permission to post to the target channel.

### Error Messages

- `"Slack webhook not configured"` -- Set `slack_webhook` in Settings > Integrations.
- `"Slack bot token not configured"` -- Set `slack_bot_token` in Settings > Integrations.
- `"Teams webhook not configured"` -- Set `teams_webhook` in Settings > Integrations.
- `HTTP 404` on Teams -- The webhook URL has expired or the connector was removed. Recreate it.
- `HTTP 403` on Slack -- The webhook was revoked. Generate a new one in the Slack App settings.
